# Ori's Devlog

Astro와 Starlight 프레임워크로 구축된 개인 기술 블로그입니다. 대규모 포스팅을 고려하여 사이드바와 카테고리 목록을 블로그형 인터페이스로 구성했습니다.

## 🛠️ 기술 스택
- **Framework:** Astro & Starlight
- **Language:** TypeScript
- **Package Manager:** npm

---

## 🚀 1. 로컬 개발 서버 실행

로컬 환경에서 개발 서버를 열고 블로그 포스팅 목록 및 개별 글들을 실시간으로 확인하려면 아래 명령어를 실행합니다.

```bash
# 의존성 설치 (최초 1회 또는 패키지 추가 시)
npm install

# 개발 서버 실행
npm run dev
```

서버가 구동되면 브라우저에서 `http://localhost:4321`에 접속하여 블로그를 탐색할 수 있습니다.

---

## 📦 2. 정적 빌드 검증

배포하기 전에 로컬에서 정적 빌드가 정상적으로 완료되는지 테스트할 수 있습니다.

```bash
npm run build
```

빌드가 성공하면 정적 아웃풋 파일들이 `dist/` 폴더에 생성됩니다.

## ✍️ 3. 블로그 포스팅(글 추가) 작성 가이드

### 3.1 신규 글 추가
새 포스팅은 `src/content/docs/[카테고리명]/` 디렉토리에 마크다운(`.md`) 또는 MDX(`.mdx`) 파일로 작성합니다.

- **cs**: 컴퓨터 공학 관련 기초 지식 정리
- **unity**: 유니티 엔진 기능 또는 개념 관련 정리
- **ps**: 코딩 테스트 문제 풀이 기록, 자료구조, 알고리즘 등에 대한 개념 정리
- **etc**: 미분류 게시물

예를 들어 `unity` 카테고리에 새 글을 추가하고 싶다면:
1. `src/content/docs/unity/my-new-post.md` 파일을 생성합니다.
2. 파일 최상단(Frontmatter)에 다음과 같이 메타데이터를 기재합니다.

```markdown
---
title: 나의 새로운 분석 글
description: 이 포스팅에 대한 간단한 한 줄 요약
tags: [Unity, Optimization]
date: 2026-06-07
---

## 내용
여기에 마크다운 문법으로 글 내용을 자유롭게 작성합니다.
```

- **tags**: 포스팅에 적용할 태그를 배열 형태로 지정합니다. 이 태그들은 빌드 시 자동으로 집계되어 `/tags` 및 `/tags/[태그명]` 페이지에 정적 반영됩니다.
- **date**: `YYYY-MM-DD` 형식으로 포스팅 작성일을 기재합니다. 카테고리 인덱스 페이지에서 최신 글 정렬(역순)의 기준이 됩니다.

### 3.2 카테고리 자동화 구조
- **좌측 사이드바**: 개별 글 목록 대신 **카테고리 대메뉴(CS, Unity, PS, 기타)**만 노출되어, 글이 수백 개 쌓여도 사이드바가 무한히 길어지지 않고 깔끔함을 유지합니다.
- **카테고리 인덱스**: 각 카테고리 루트는 동적 렌더링 컴포넌트(`CategoryList.astro`)를 내장하고 있습니다. 사용자가 폴더 아래에 마크다운 글만 추가하면, 해당 카테고리 게시글 목록에 **최신 작성일 순으로 자동 리스트업**됩니다.

## 🌐 4. 배포 방식

이 프로젝트는 GitHub Actions를 통해 자동으로 빌드 및 배포됩니다.

1. 로컬에서 변경된 코드를 커밋한 뒤 `main` 브랜치로 푸시(Push)합니다.
2. GitHub 저장소의 `Actions` 탭에서 **Deploy to GitHub Pages** 워크플로우가 트리거되어 빌드 및 배포를 수행합니다.
3. 배포 프로세스가 성공하면 배포 주소 [orikwon-dev.github.io](https://orikwon-dev.github.io)에 즉시 반영됩니다.

> 💡 **최초 배포 설정:** GitHub 저장소 설정(`Settings > Pages`)에서 배포 소스(Build and deployment > Source)가 **GitHub Actions**로 지정되어 있는지 반드시 확인해 주세요.
